## рҹҺҜ What is TrapNet XDR?

TrapNet XDR is a **full-stack cyber deception platform** that simulates vulnerable services (SSH, HTTP login panels) to lure and capture attacker behavior, analyze threats using machine learning, and visualize everything in a real-time multi-page dashboard.

```
Attacker arrives вҶ’  Fake SSH / HTTP panel вҶ’  Credentials captured
                                           вҶ’  Commands logged
                                           вҶ’  ML risk score assigned
                                           вҶ’  MITRE ATT&CK mapped
                                           вҶ’  Alert triggered
                                           вҶ’  Real-time dashboard updated
```

## рҹ§  ML Pipeline

```
Raw Session Data (captured from honeypot)
    в”Ӯ
    в–ј
Feature Extraction (8 features per session)
  в”ңв”Җ login_attempts        вҶ’ number of login tries
  в”ңв”Җ unique_usernames      вҶ’ credential spray indicator
  в”ңв”Җ unique_passwords      вҶ’ dictionary attack indicator
  в”ңв”Җ commands_executed     вҶ’ post-auth activity count
  в”ңв”Җ dangerous_commands    вҶ’ wget/curl/rm -rf/shadow etc.
  в”ңв”Җ session_duration_s    вҶ’ time-on-target
  в”ңв”Җ request_rate          вҶ’ attempts per minute
  в””в”Җ entropy_score         вҶ’ Shannon entropy of credentials
    в”Ӯ
    в–ј
StandardScaler (normalize all features to z-scores)
    в”Ӯ
    в–ј
Isolation Forest (200 trees, contamination=0.05)
    в”Ӯ
    в”ңв”Җ Anomaly Score вҶ’ normalized to 0вҖ“100
    в”ңв”Җ Prediction: ANOMALY (-1) or NORMAL (1)
    в””в”Җ Risk Level: CRITICAL / HIGH / MEDIUM / LOW

Model Performance:
  Accuracy   ~95.2%
  Precision  93.2%
  Recall     94.7%
  Type       Unsupervised (no labels needed)
```

## рҹ”’ Security Model

- SSH/HTTP services are **fully simulated in software** вҖ” no real OS access
- All credentials submitted are **logged and rejected** вҖ” always returns 401
- Shell responses are **hardcoded fake outputs** вҖ” no real command execution
- Rate limiting prevents the honeypot itself from being abused
- MongoDB injection prevented via parameterized queries
